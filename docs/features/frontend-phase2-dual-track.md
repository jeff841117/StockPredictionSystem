# 前端第二階段雙路線文件

## 路線定案

本輪已正式定案：

- `main = 作品集版第二階段主線`
- `daily-use = 未來分支方向`

因此後續前端第二階段的主線規劃與開發，應優先服務以下目標：

1. 展示力
2. 研究感
3. 產品完成度
4. 面試與作品集敘事清楚度

`daily-use` 相關候選功能保留作為未來分支方向，不納入目前主線開發排序。

## 目前主線已完成成果

截至目前，`main` 這條作品集版第二階段主線已完成的前端成果為：

1. 個股頁 `Research Summary` 區
2. 首頁 `Screener / Research Candidates` 展示區
3. 首頁與個股頁的研究工作台版位優化
4. 個股頁圖表疊層切換
5. Demo 用投資組合摘要強化

若要做總驗收，請優先參考：

- [docs/features/frontend-phase2-acceptance-summary.md](frontend-phase2-acceptance-summary.md)

## 文件目的

本文件整理前端第二階段的候選方向，刻意拆成兩條路線：

1. 作品集版
2. 日常使用版

本文件的主要用途，是保留雙路線判斷依據與後續候選方向；它不再是目前驗收的主要入口文件。

## 目前前端基底

### 首頁

- 目前語彙已是 `Stock Screener Workspace`
- 已有查詢主區、研究流程、Screener 展示與 Portfolio preview
- 核心可用功能仍是單一股票查詢 + 自訂日期區間

### 個股頁

- 目前語彙已是 `Research Workspace`
- 已有 `Research Summary`
- 已有價格圖、圖表疊層切換、歷史資料表與右側操作欄
- 已有作品集導向的 `Gap Review` 與研究流程導引
- 已有虛擬資金摘要、技術線圖與研究導覽語氣

## 拆線原則

### `main` 定位

- 保留作品集展示力
- 保留通用研究能力
- 不太早變成高度個人化工具
- 優先做「展示感高、低耦合、面試容易說明」的功能

### `daily-use` 分支定位

- 提升查找效率與連續操作流暢度
- 放高頻操作、個人偏好、快捷流程
- 可以容納較重的狀態管理與個人化設計
- 目前不納入主線開發

## 標示規則

- `展示感加分`：
  - 高：明顯提升作品集說服力
  - 中：有幫助，但不是最亮點
  - 低：主要是效率改善
- `系統影響`：
  - 低：大多是前端呈現或小幅資料整理
  - 中：會牽動查詢流程、狀態管理或頁面資料組裝
  - 高：會影響未來真實交易系統的資料流、權限、持股或操作模型
- `建議落點`：
  - `main`
  - `daily-use`
  - `兩邊都可`

## 主線後續候選順序

### 作品集版後續候選順序

在已完成目前這一輪主線成果後，若要繼續往下延伸，可優先考慮：

1. Workspace 分頁內容落地
2. Research Summary 區
3. 首頁功能路線圖卡片
4. 圖表觀察標註

原因：

- 展示感高
- 不必先改交易核心規則
- 最能強化「研究工具 + 模擬交易系統」完成度

### `daily-use` 未來分支候選順序

優先做：

1. 個股頁快速切換日期區間
2. 最近查詢紀錄
3. 常用代號快速填入
4. 查詢失敗重試 UX

原因：

- 能先提升高頻操作效率
- 不會太早碰真實交易系統的高耦合狀態
- 可先保留交易核心不變

## 哪些功能建議先留在 `main`

建議優先留在 `main` 的，是這類功能：

1. 展示力高
2. 研究感強
3. 對資料流影響小
4. 面試時容易解釋價值

目前最適合留在 `main` 的候選：

1. 首頁功能路線圖卡片
2. Research Summary 區
3. Workspace 分頁內容落地
4. 圖表觀察標註
5. 資料可信度與限制說明

## 哪些功能建議放到未來 `daily-use` 分支

建議放到 `daily-use` 分支的，是這類功能：

1. 強個人化
2. 高頻操作優化
3. 會逐步牽動狀態管理
4. 可能影響未來真實交易系統流程

目前最適合放到 `daily-use` 的候選：

1. 最近查詢紀錄
2. 預設日期區間快捷鈕
3. 買進表單快速複用
4. 個人化觀察標記
5. 收藏 / 持股 / 交易狀態整合
6. 同頁 SELL 入口

## 不建議第二階段太早做的項目

1. 直接做完整多條件 screener 後端
2. 直接做重度個人化 dashboard
3. 直接把個股頁改成真實交易工作台
4. 提前做會綁死真實券商流程的操作模型

原因：

- 這些看似前端，其實很容易牽動資料模型、狀態同步與未來交易設計
- 容易讓 `main` 過早偏向個人工具，而不是作品集與通用研究系統

## 交接給後續規劃 / 開發的建議

1. 目前主線已定案為作品集版，不再雙線並行
2. 前端第二階段目前已有一份總驗收入口文件，驗收時應優先使用該文件
3. `daily-use` 類功能先保留在文件，不進入目前 `main` 驗收範圍
4. 若進入下一輪主線開發，仍建議維持一次一個小功能交付
