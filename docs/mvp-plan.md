# 第一版 MVP 功能拆解與開發順序

## 一、第一版 MVP 核心目標

第一版完成時，應該至少能做到：

1. 使用者輸入台股代號
2. 系統顯示股票基本資訊
3. 系統顯示歷史股價資料
4. 系統顯示走勢圖
5. 使用者可以加入收藏
6. 使用者可以模擬買進與賣出
7. 系統可以顯示目前持股、成本、損益
8. 系統可以顯示交易紀錄
9. 系統可以在本機跑起來並對外展示

## 二、MVP 功能模組拆解

### 模組 1：專案基礎架構

目標：  
先把專案骨架建立起來，讓後續功能有地方放。

內容：

1. 建立 Python 專案資料夾結構
2. 建立虛擬環境
3. 安裝基本套件
4. 建立 FastAPI 主程式
5. 建立基本首頁或測試頁
6. 設定 `.env` 與基本設定檔
7. 建立 requirements 檔案

這階段重點不是功能，而是讓專案可以啟動。

複習重點：

1. Python 專案結構
2. 套件安裝
3. import 模組
4. 基本 FastAPI 啟動方式

完成標準：

1. 本機可啟動 FastAPI
2. 瀏覽器可看到首頁或 API 測試畫面

### 模組 2：股票資料查詢

目標：  
輸入股票代號後，能取得基本股票資料與歷史價格。

內容：

1. 選定資料來源
2. 撰寫股票資料抓取函式
3. 處理 API 回傳格式
4. 顯示股票名稱、代號、日期、開高低收量
5. 先用單一股票測試

建議先從以下兩件事開始，不要一開始就做太多查詢條件：

1. 查單一股票
2. 查固定日期區間

複習重點：

1. requests
2. JSON 資料解析
3. pandas DataFrame
4. 函式拆分
5. try/except

完成標準：

1. 輸入股票代號可以成功取得資料
2. 畫面或 API 能顯示整理後的資料

### 模組 3：歷史資料顯示與圖表

目標：  
把抓到的資料用表格與圖表呈現。

內容：

1. 顯示歷史股價表格
2. 顯示收盤價折線圖
3. 提供近 7 / 30 / 90 天切換
4. 視情況加入 K 線圖

建議順序：  
先做折線圖，再考慮 K 線圖，因為折線圖比較容易先完成。

複習重點：

1. pandas 欄位處理
2. 日期格式轉換
3. Plotly 或 Matplotlib
4. 前端模板資料呈現

完成標準：

1. 可看到歷史資料表格
2. 可看到至少一種股價趨勢圖

### 模組 4：技術指標

目標：  
加入最基本的分析資訊，讓系統不像只是查價工具。

第一版只建議做：

1. MA5
2. MA20

如果時間夠，再加：

1. 成交量變化
2. 日報酬率

原因：  
均線最適合當第一個技術指標，容易實作，也容易展示。

複習重點：

1. pandas rolling
2. 新欄位計算
3. 資料缺值處理
4. 基本商業邏輯

完成標準：

1. 圖表或表格中可顯示 MA5、MA20
2. 可解釋這些欄位是如何算出來的

### 模組 5：收藏清單

目標：  
讓使用者可以收藏想關注的股票。

內容：

1. 建立收藏資料表
2. 提供加入收藏功能
3. 提供移除收藏功能
4. 顯示收藏清單

這是很適合展示 CRUD 的模組。

複習重點：

1. SQLite 或 PostgreSQL 基本操作
2. INSERT / SELECT / DELETE
3. FastAPI 路由設計
4. 表單送出或按鈕操作

完成標準：

1. 可以加入收藏
2. 可以查看收藏清單
3. 可以刪除收藏

### 模組 6：模擬買進與賣出

目標：  
完成第一版最有作品集價值的核心互動功能。

內容：

1. 設定初始虛擬資金
2. 輸入股票代號、數量、價格
3. 模擬買進
4. 模擬賣出
5. 記錄每一筆交易
6. 更新持股狀態

這裡先不要做太複雜的手續費、交易稅、零股、當沖邏輯。  
第一版先把基本交易流程做通。

複習重點：

1. 條件判斷
2. 函式設計
3. CRUD
4. 資料一致性
5. 商業邏輯拆分

完成標準：

1. 可以成功新增買入交易
2. 可以成功新增賣出交易
3. 不會出現賣出超過持股的情況
4. 交易資料有被保存

### 模組 7：持股與損益顯示

目標：  
根據交易紀錄計算目前持股狀態。

內容：

1. 顯示目前持股數量
2. 顯示平均成本
3. 顯示當前市值
4. 顯示未實現損益
5. 顯示已實現損益
6. 顯示總交易紀錄

這是面試時很值得講的模組，因為它能展示你有處理狀態與計算邏輯。

複習重點：

1. 資料彙總
2. groupby
3. 邏輯驗證
4. 查詢與計算分離

完成標準：

1. 可看到每檔股票目前持股
2. 可看到平均成本與損益
3. 可看到完整交易紀錄

### 模組 8：基本部署

目標：  
讓系統能從外部開啟，作為作品集展示。

內容：

1. 確認本機啟動方式
2. 設定 Cloudflare Tunnel
3. 測試外部連線
4. 整理啟動與部署說明

複習重點：

1. 執行環境
2. 啟動指令
3. 環境變數
4. 部署流程紀錄

完成標準：

1. 本機可正常啟動
2. 外網可透過 Cloudflare 開啟系統
3. 有基本部署文件

## 三、建議開發順序

我建議照這個順序做：

1. 專案基礎架構
2. 股票資料查詢
3. 歷史資料顯示與折線圖
4. 技術指標 MA5 / MA20
5. 收藏清單
6. 模擬買進與賣出
7. 持股與損益顯示
8. 基本部署
9. README 與文件整理

這個順序的好處是：

1. 前面先建立成就感
2. 中段開始碰資料庫與 CRUD
3. 後段再做邏輯較重的交易功能
4. 最後再處理部署，避免太早被環境問題卡住

## 四、每階段預估成果

### 第 1 階段成果

你會有一個可以啟動的 Python Web 專案。

### 第 2 階段成果

你會有一個能查詢台股資料的小系統。

### 第 3 階段成果

你會把資料成功變成可閱讀的畫面與圖表。

### 第 4 階段成果

你會開始有分析功能，不只是純查詢。

### 第 5 到 7 階段成果

你會完成作品集最重要的互動功能，也就是模擬交易與損益展示。

### 第 8 到 9 階段成果

你會把專案從「能寫」變成「能展示」。

## 五、建議第一版先不要做的功能

1. 真實下單
2. 使用者登入
3. 多人帳號系統
4. 複雜回測
5. 即時串流報價
6. 深度學習模型
7. 太多技術指標
8. 過度漂亮但複雜的前端

原因不是這些不重要，而是它們會大幅提高第一版失敗率。

## 六、你和 AI 的最佳合作方式

很建議你開發時用這種節奏：

1. 先決定這一輪只做一個模組
2. 先問 AI 這個模組要學會哪些觀念
3. 你先自己寫第一版
4. 再讓 AI 幫你檢查錯誤與重構方向
5. 最後再補筆記與整理學習紀錄
