# 台股分析與模擬交易系統 事前規格書 v1

## 1. 專案名稱

**中文名稱**：台股分析與模擬交易系統  
**英文名稱**：Taiwan Stock Analysis & Paper Trading System

## 2. 專案定位

本專案是一個以 **Python 為主要開發架構** 的台灣股票查詢、分析與模擬交易網頁系統。  
目標是建立一個可展示資料處理、後端 API、資料庫設計、基礎機器學習應用與部署能力的作品集專案。

本專案第一階段不以真實自動下單為目標，而是優先完成：

1. 台股資料查詢
2. 歷史資料視覺化
3. 技術指標分析
4. 模擬買賣
5. 基礎預測功能
6. 文件化與部署展示

## 3. 專案目標

### 3.1 主要目標

建立一套可在瀏覽器中操作的台股分析系統，讓使用者可以：

1. 查詢台灣股票歷史資料
2. 觀看價格走勢與基礎圖表
3. 查看簡單技術分析結果
4. 使用虛擬資金進行模擬交易
5. 查看交易紀錄與損益
6. 體驗基礎 AI 或機器學習預測功能
7. 從外網透過 Cloudflare 存取部署後的系統

### 3.2 作品集目標

本專案希望展示以下能力：

1. Python 基礎與模組化開發能力
2. API 串接與資料清洗能力
3. Pandas / NumPy 資料處理能力
4. Web 後端開發能力
5. 資料庫設計與 CRUD 能力
6. 基礎機器學習應用能力
7. 系統部署與文件撰寫能力

## 4. 專案背景與限制

### 4.1 背景

開發者目前為 Python 與 AI 初學者，已學過：

1. Python 基礎語法
2. 函式、迴圈、變數
3. 基礎資料處理
4. NumPy、Pandas
5. 爬蟲相關內容

### 4.2 限制條件

1. 部分已學內容已遺忘，需在開發過程中同步複習
2. 專案需作為求職作品集，不適合直接由 AI 完整代做
3. 股票金融知識目前不是開發中的優先教學內容
4. 專案核心需以 Python 為主
5. 專案需求與內容後續可能持續調整
6. 預計部署於個人電腦伺服器，並透過 Cloudflare 對外提供存取

## 5. AI 協作原則

本專案與 AI 的合作方式如下：

1. AI 以教學輔助、結構引導、除錯協助、規劃建議為主
2. 不直接一次產出完整最終成品
3. 開發流程以模組化、小步驟、可理解方式進行
4. 每完成一個模組時，應搭配對應複習重點
5. 若使用者卡住，可提供局部範例、解題方向與修正建議
6. 金融知識可先不在主開發流程深入說明，但需於後期整理成獨立文件

## 8. 系統核心功能說明

### 8.1 股票資料查詢

功能目標：  
輸入股票代號後，取得對應股票的歷史資料與基本資訊。

預期內容：

1. 股票代號
2. 股票名稱
3. 日期
4. 開盤價
5. 最高價
6. 最低價
7. 收盤價
8. 成交量

### 8.2 股價視覺化

功能目標：  
將歷史資料以圖表方式呈現，方便觀察趨勢。

預期內容：

1. 收盤價折線圖
2. K 線圖
3. 區間切換
4. 技術指標疊圖

### 8.3 技術指標分析

功能目標：  
提供基礎技術分析資訊，作為觀察用途。

第一版建議：

1. MA5
2. MA20
3. 成交量變化

第二版可考慮：

1. RSI
2. MACD
3. 布林通道

### 8.4 模擬交易

功能目標：  
使用虛擬資金進行股票買賣練習。

預期內容：

1. 初始虛擬資金設定
2. 模擬買入
3. 模擬賣出
4. 平均成本計算
5. 未實現損益
6. 已實現損益
7. 交易紀錄查詢

### 8.5 預測功能

功能目標：  
展示基礎機器學習應用能力，不作為真實投資依據。

第一版建議形式：

1. 預測隔日漲跌方向
2. 顯示模型準確率
3. 顯示使用的特徵欄位
4. 顯示模型限制說明

功能定位說明：  
本功能僅作為資料分析與機器學習練習展示，不構成投資建議。

## 9. 技術架構規劃

### 9.1 主要技術

1. Python
2. FastAPI
3. Pandas
4. NumPy
5. SQLite 或 PostgreSQL
6. Plotly 或 Matplotlib
7. Scikit-learn

### 9.2 前端方案建議

優先建議方案：

1. FastAPI + Jinja2 + Bootstrap

理由：

1. 架構較單純
2. 對初學者較友善
3. 可把重點放在 Python 與系統邏輯

備選方案：

1. Streamlit
2. React + FastAPI

不建議一開始採用 React 的原因：  
前後端分離會增加學習與開發負擔，可能影響專案完成度。

### 9.3 部署方式

1. 系統執行於本機電腦伺服器
2. 使用 Cloudflare Tunnel 對外提供連線
3. 視需求搭配反向代理與網域設定

## 10. 資料來源規劃

### 10.1 優先考慮資料來源

1. TWSE OpenAPI
2. FinMind

### 10.2 選擇原則

1. 優先官方或穩定來源
2. 優先適合 Python 串接
3. 優先資料欄位清楚、格式一致
4. 優先合法且適合作品集展示

### 10.3 注意事項

1. 不同來源資料格式可能不同
2. 歷史資料完整度需驗證
3. 後續若更換來源，需注意欄位轉換與一致性
